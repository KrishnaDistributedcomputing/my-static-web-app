Azure Traffic Manager Flashcards

Test your knowledge about Azure Traffic Manager with these flashcards. Click on a card to reveal the answer.

What is Azure Traffic Manager?

Azure Traffic Manager is a DNS-based traffic load balancer that improves the performance and availability of applications.

At which layer does Traffic Manager operate?

Traffic Manager operates at the DNS level (Application layer, Layer-7).

What are the traffic-routing methods supported by Traffic Manager?

Performance, Priority, Geographic, and Weighted methods.

How does Traffic Manager ensure endpoint availability?

Traffic Manager continuously monitors the health of endpoints and performs automatic failovers when endpoints fail.

What are Traffic Manager profiles used for?

Profiles manage traffic and contain settings for endpoints and routing methods.

What types of endpoints does Traffic Manager support?

Azure, external, or nested endpoints.

How should vanity domain names be configured with Traffic Manager?

Vanity domain names must point to the Traffic Manager domain name using CNAME records.

What role do health checks play in Traffic Manager?

Health checks determine the current health of each endpoint, influencing traffic routing decisions.

What is the recommended TTL for DNS records in Traffic Manager?

The recommended TTL is 60 seconds for quick failover.

How does Traffic Manager ensure only healthy endpoints receive traffic?

Through continuous endpoint monitoring.

How does Traffic Manager handle endpoint failures?

It performs automatic failovers to the next best available endpoint.

What is the purpose of Geographic routing in Traffic Manager?

To direct users to endpoints based on their geographic location, ensuring compliance and performance.

How does Weighted routing work in Traffic Manager?

It distributes traffic based on assigned endpoint weights, allowing for proportional traffic distribution.

What does Priority routing in Traffic Manager do?

It sends traffic to the highest priority endpoint available, with lower priority endpoints as backups.

What is Performance routing in Traffic Manager?

It directs traffic to the endpoint with the lowest latency, improving user experience.

What are the steps to set up Traffic Manager?

Create a profile, add endpoints, and configure routing methods based on application requirements.

How can Traffic Manager configurations be managed?

Using Azure CLI commands for flexibility and automation.

What is the benefit of using Azure DNS with Traffic Manager?

For seamless integration and simplified management.

What are the management options for Traffic Manager profiles?

Profiles can be enabled, disabled, or deleted as needed.

Why is it important to test Traffic Manager configurations?

To ensure proper functionality and performance with multiple clients from various locations.

How does DNSSEC enhance security in Traffic Manager?

DNSSEC protects against DNS spoofing attacks by ensuring DNS responses are authentic.

What should be ensured for secure endpoint communication?

Ensure endpoints use HTTPS.

How can Traffic Manager be integrated for continuous security monitoring?

Integrate with Azure Security Center.

How should permissions for Traffic Manager resources be managed?

Use Azure Role-Based Access Control (RBAC).

What should be enabled to monitor Traffic Manager activities?

Enable diagnostic logging and metrics.

How can access to Traffic Manager endpoints be restricted?

Implement IP restrictions on endpoints.

What type of certificates should be used for Traffic Manager endpoints?

Use SSL/TLS certificates to encrypt data in transit.

Which Azure service can be integrated with Traffic Manager for added security?

Integrate with Azure Firewall.

What security practices should be followed for Traffic Manager?

Follow Azure security best practices, including updates, assessments, and testing.

What compliance standards should Traffic Manager configurations adhere to?

Ensure compliance with industry standards like GDPR, HIPAA, and ISO/IEC 27001.
