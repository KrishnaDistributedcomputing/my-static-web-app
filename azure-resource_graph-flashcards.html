Azure Resource Graph Flashcards

Test your knowledge about Azure Resource Graph with these flashcards. Click on a card to reveal the answer.

What is Azure Resource Graph?

Azure Resource Graph is an Azure service designed to extend Azure Resource Management by providing efficient and performant resource exploration with the ability to query at scale across a given set of subscriptions.

What are the capabilities of Azure Resource Graph queries?

Query resources with complex filtering, grouping, and sorting by resource properties; explore resources iteratively based on governance requirements; assess the effect of applying policies; query changes made to resource properties.

What are the basic resource fields supported by Azure Resource Graph?

Resource name, ID, Type, Resource Group, Subscription.

How is Azure Resource Graph kept current?

Azure Resource Graph updates its database regularly and whenever there is a change notified by Azure Resource Manager.

Which query language does Azure Resource Graph use?

Azure Resource Graph's query language is based on the Kusto Query Language (KQL) used by Azure Data Explorer.

How does Azure Resource Graph support Azure Lighthouse?

Azure Resource Graph supports Azure Lighthouse, enabling service providers to manage customer-delegated subscriptions and resource groups from their own tenant.

What are the steps to run your first query using Azure Resource Graph Explorer?

Sign in to the Azure portal, search for Resource Graph Explorer, select the directory and scope, enter and run the query, review the results.

How do you enable Azure CLI to query resources using Azure Resource Graph?

Install the Azure Resource Graph extension for Azure CLI using `az extension add --name resource-graph`.

What is the purpose of Resource Graph's schema browser?

To find and select resource types and properties to be used in queries, making it easier to construct and refine queries.

What are the integration options for creating alerts with Azure Resource Graph?

Azure Resource Graph queries can be integrated with Azure Monitor and Log Analytics to create alert rules based on resource conditions.

How does Azure Resource Graph handle large datasets?

Azure Resource Graph uses efficient indexing and querying mechanisms to handle large datasets and provides throttling to ensure optimal performance.

What is a skip token in Azure Resource Graph?

A skip token is used to handle pagination in query results, allowing retrieval of more than the default limit of records in a single query.

What are some common use cases for Azure Resource Graph?

Common use cases include inventory management, resource governance, security and compliance monitoring, and cost management.

How can Azure Resource Graph queries be shared?

Azure Resource Graph queries can be saved and shared as either private or shared queries, with shared queries being accessible to other users with appropriate permissions.

What is the maximum number of records returned by Azure Resource Graph by default?

By default, Azure Resource Graph returns a maximum of 1,000 records per query.

How can Azure Resource Graph be integrated with Power BI?

Azure Resource Graph can be integrated with Power BI using the Power BI connector, allowing users to run queries and visualize results within Power BI.

What are some advanced query capabilities of Azure Resource Graph?

Advanced query capabilities include joining multiple tables, using functions for aggregations, and implementing complex filtering logic.

What is the purpose of the Azure Resource Graph Explorer?

Azure Resource Graph Explorer is a tool in the Azure portal that provides a user-friendly interface for constructing, running, and visualizing Resource Graph queries.

How does Azure Resource Graph enhance Azure Policy?

Azure Resource Graph enhances Azure Policy by enabling large-scale policy compliance assessments and providing insights into resource configurations and compliance states.

What is the benefit of using Azure Resource Graph for resource governance?

Azure Resource Graph allows organizations to effectively manage and govern their Azure resources at scale, ensuring compliance and optimizing resource utilization.
