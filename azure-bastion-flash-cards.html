What is Azure Bastion?

A managed PaaS that provides secure RDP and SSH access to VMs.

Do you need a public IP on the VM to connect via Azure Bastion?

No, Bastion connects to VMs using their private IP addresses.

What protocol does Azure Bastion use for connections?

RDP/SSH connections over TLS (port 443).

Can Azure Bastion be used with virtual network peering?

Yes, it supports connecting to VMs in peered virtual networks.

Which SKU supports custom inbound ports?

Standard SKU.

What is the minimum subnet size for AzureBastionSubnet with host scaling?

/26 or larger.

Which authentication methods are supported for SSH connections to Windows VMs?

Username/password, SSH keys from a local file, password from Azure Key Vault.

How many concurrent RDP connections can a single Azure Bastion instance support for medium workloads?

40 concurrent RDP connections.

Which feature is NOT supported in Azure Bastion private-only deployments?

Connections to the bastion host via public IP address.

Which SKU is required for session recording?

Standard SKU.

What is the maximum number of Bastion hosts in a virtual network?

50 hosts.

What is a limitation of the az network bastion tunnel command?

It does not support file downloads from the target VM.

How does Azure Bastion enhance security?

By eliminating the need for public IP addresses on VMs and encrypting traffic over TLS.

Which Azure Bastion feature allows seamless RDP/SSH from the Azure portal?

HTML5-based web client.

Does Azure Bastion require an agent or special client software on VMs?

No, it operates without the need for any agent or special client software on your VMs.

What types of VMs can Azure Bastion connect to?

Azure Bastion can connect to both Linux and Windows VMs.

Does Azure Bastion support multi-factor authentication (MFA)?

Yes, Azure Bastion supports MFA for enhanced security.

How does Azure Bastion simplify network security management?

It eliminates the need to manage NSGs for RDP/SSH traffic.

Can you deploy Azure Bastion in a hub-and-spoke network architecture?

Yes, Azure Bastion supports deployment in a hub-and-spoke network architecture.

What is the default port used by Azure Bastion?

Azure Bastion uses port 443 for RDP/SSH connections.

Is Azure Bastion a fully managed service?

Yes, Azure Bastion is a fully managed PaaS offering.

Can Azure Bastion be used to connect to VMs in a peered virtual network?

Yes, Azure Bastion can connect to VMs in peered virtual networks.

Does Azure Bastion support network security groups (NSGs)?

Yes, you can use NSGs to control traffic to the AzureBastionSubnet.

What is a key advantage of using Azure Bastion?

It enhances security by eliminating the need for public IP addresses on VMs.

Can Azure Bastion be integrated with Azure AD?

Yes, Azure Bastion can be integrated with Azure AD for identity management.

Is session logging supported in Azure Bastion?

Yes, session logging is supported for auditing and compliance.

What is the recommended subnet size for Azure Bastion?

The recommended subnet size is /27 or larger.

Does Azure Bastion support just-in-time (JIT) VM access?

Yes, Azure Bastion supports JIT VM access for enhanced security.

Can Azure Bastion connect to VMs in different regions?

No, Azure Bastion can only connect to VMs in the same region.

What type of encryption does Azure Bastion use for connections?

Azure Bastion uses TLS encryption for all connections.
